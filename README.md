# NAFuse
NAFuse v.1.1 (Nucleic Acid Fusion) is an integrated methodology that combines and compares the output of a gene fusion detector with that of the BAM-file generated by whole genome sequencing. This allows for the detection of breakpoints in both partner genes/regions on both the RNA and DNA levels without having to manually search output files generated during downstream processing of BAM-files. The main idea of this method is to help to identify, among the huge amount of gene fusions detected with RNA-seq, new gene fusions with DNA-level support.

When using the algorithm, please cite: 

### Prerequisites

Before starting the analysis, make sure to have the prerequisites.

1.  BEDTools Suite (v. 2.26.0 or later).
2.  BEDOPS tools (2.4.32 or later).
3.  Gene annotation. 
    Based on the genome reference, download the matching file with gene coordinates and save it in BED format. Having the reference in BED format is essential to run the bedmap command.
    
### Usage
 
#### Preparation file for NAFuse
  
1. Create subfolders in the NAFuse directory
 ```
mkdir Bedfiles
mkdir Annotated_bedfiles
mkdir DNA_seq
mkdir RNA_seq
mkdir NAFuse_output
 ```
2. Prepare the RNA-sequencing file. 
From the outputof a gene fusion detector create a text file with two columns. The first column containing the 5' partner and the second column containing the 3' partner. Each line now represents a gene fusion. Save this file in folder RNA_seq/.
 
3. Convert the BAM file in BED file with bamToBed sub-command
 ```
bamToBed -i $file >  NAFuse/Bedfiles/{filename}.bed
 ```
 
4. Create a gene annotation file.
	 It's possible to find a file with gene annotation in folder Gene_annotation_files. If you want to create your own gene annotation file make sure that this file       is sorted and tab delimited.
   
5. Sort and annotate the BED file optained in 3
```
sort-bed --max-mem 2G NAFuse/Bedfiles/$file | bedmap --echo --echo-map-id --delim '\t' --multidelim '\t' - NAFuse/Gene_annotation_files/ >  NAFuse/DNA_seq/{filename}.bed
```

6. Split the file based on the number of the read end.
```
#Read1
awk '{gsub("/","\t",$0);print;}' $file | awk '$5=="1"' | cut -f 4- | awk '{gsub(" ","\t",$0);print;}' > NAFuse/DNA_seq/{filename}_read1.bed

#Read2
awk '{gsub("/","\t",$0);print;}' $file | awk '$5=="2"' | cut -f 4- | awk '{gsub(" ","\t",$0);print;}' > NAFuse/DNA_seq/{filename}_read2.bed
```

#### NAFuse core

The bed file generated in point 3 contains rows with read information. It is important to notice that, when the mate information is available, each read name ends with “/1” or “/2” (readname/1 or readname/2) and that they identify the two ends of one read. The two ends can align in the same gene (Figure 1A), in two different genes but on the same chromosome (Figure 1B) and in two different genes and chromosomes (Figure 1C).


![NAFuse](https://user-images.githubusercontent.com/91432163/222132567-1545c34a-2d72-4f6d-afb3-bcefefb8f614.jpg)


7. It’s time to run the core of NAFuse. The script is written in Python3 and it can be found in bin/ folder called NAFuse.py. If you want to analyze multiple files, run NAFuse_forloop.py. The script operates in two main steps.

   -	Search if there are matches between the output file from a gene fusion detector generated in point 2 and the genomic files generated in point 6.
   -	Create a file where each row contains the information belonging to the same read. It’s possible to merge the lines based on read name. If a gene fusion is present in a line, it’ll be considered a potential gene fusion.

#### Data Analysis
You can now analyze you files and find new gene fusions.

The results file will be saved in /NAFuse/NAFuse_output/ directory.
The resulting file cointains those fusions present in both DNA and RNA. 
We suggested to manually check the reality of the gene fusions by analysing the exon coverage from RNA sequencing and the brekpoint on DNA level. The fusion sequence can be analyzed with ORF finder (https://www.ncbi.nlm.nih.gov/orffinder/).

### Validation and Tips

-	NAFuse was tested in cases reported in this study. It was able to successfully detect all the gene fusions described when DNA and RNA were available, such as FOSL1::RELA and FUS::TFCP2.
-	In some instances, the breakpoint on the DNA level could be located upstream of the 3’ partner gene. To solve this, we increased gene boundaries (start and stop) by 50 000 bp each.
-	Use the correct gene annotation file. Be sure to use the same gene annotation used by your gene fusion detector of choice. It can be necessary to merge multiple gene annotations files (GRCh37/hg19 and GRCh38/hg38).
